Binário:

O código binário é uma linguagem de máquina. Ele é usado para que as máquinas possam interpretar as informações que são enviadas por meio dos comandos – como um texto digitado no teclado, por exemplo, que se transforma em uma letra. Enquanto existem diversas linguagens de programação, o código binário é um só.

Vantagens:

O sistema binário é simples e eficiente, usando apenas 0 e 1 para representar dados. Isso facilita a conversão de dispositivos físicos em elementos computacionais, com baixo risco de erro na interpretação dos dados. Essa simplicidade é crucial para dispositivos eletrônicos e computadores.

Desvantagens:

Uma desvantagem do sistema de numeração BCD é que 4 bits precisam ser utilizados para representar qualquer número decimal.

Onde o código binário é usado?

Tudo que contém um chip programado, desde carros, drones e aviões até bonecos, usa linguagem binária.

Assembly:

Assembly ou linguagem de montagem é uma notação legível por humanos para o código de máquina que uma arquitetura de computador específica usa, utilizada para programar códigos entendidos por dispositivos computacionais, como microprocessadores e microcontroladores.

É eficiente?

Sim, é trabalhoso.
Sim, é demorado.
Sim, é, algumas vezes, entediante. Mas uma vez feito, e bem feito, você terá feito um ótimo trabalho e poderá ter a certeza que vai tirar o máximo de sua máquina.

Linguagens Compiladas:

São convertidas diretamente na máquina em um código de máquina que o processador pode executar. Como resultado, elas tendem a ser mais rápidas e mais eficientes em sua execução do que as linguagens interpretadas. Elas também dão ao desenvolvedor mais controle sobre alguns aspectos do hardware, como o gerenciamento da memória e o uso da CPU.

As linguagens compiladas necessitam de uma etapa de "build" (montagem) – elas precisam, primeiramente, ser compiladas manualmente. Você precisa "remontar" o programa sempre que precisar fazer uma alteração

Exemplos de linguagens compiladas puras são o C, o C++, o Erlang, o Haskell, o Rust e o Go.

Linguagens Interpretadas:

Os interpretadores passam por um programa linha por linha e executam cada comando. Aqui, se o autor decidir que quer usar um tipo diferente de óleo de oliva, só precisaria remover o antigo e adicionar o novo. Seu amigo tradutor poderia informar isso a você quando a mudança acontecesse.

Linguagens interpretadas, antigamente, eram significativamente mais lentas do que as linguagens compiladas. Porém, com o desenvolvimento da compilação just-in-time, essa distância vem diminuindo.

Exemplos de linguagens interpretadas comuns são o PHP, o Ruby, o Python e o JavaScript.

Linguagens Híbridas:

São aquelas que implementam as duas formas de leituras, compilado e interpretado, por exemplo o Java que tem um compilador que converte seu código para bytecode e na sequência isso é lido para JVM (Java Virtual Machine) que é basicamente um interpretador, para no final ser executado pelo CPU.

Aplicativos Nativos:

Como o próprio nome já diz, aplicativos nativos são desenvolvidos na linguagem exclusiva para um sistema operacional, como Android ou IOS. Por ser desenvolvido de forma exclusiva para um sistema, um aplicativo feito para IOS, por exemplo, não funcionará na plataforma Android, sendo necessário que o código seja reescrito novamente para a linguagem específica. O Aplicativo é baixado diretamente das lojas de aplicativos (App Store, Play Store ou Huawei Store), sendo necessária a aprovação das respectivas plataformas para o App ser disponibilizado aos usuários.

Aplicativos Híbridos:

O aplicativo híbrido é desenvolvido de forma que seja compatível com mais de um sistema operacional, normalmente quando falamos de desenvolvimento mobile, Android e IOS (em alguns casos Windows Phone). Ele pode ser disponibilizado para download em diversas lojas de aplicativo utilizando o mesmo código, não exigindo um download a cada atualização, entretanto seu uso, fica atrelado à uma conexão com a internet.

Low Code:

O movimento no code ou low code é uma filosofia digital que permite que qualquer pessoa crie aplicativos e programas sem a necessidade de saber programação. Isso é conhecido como "programação sem código" ou "programação visual".

Linguagens Estáticas:

Linguagens estáticas não permitem ao desenvolvedor alterar o tipo da variável depois de declarada. Geralmente a verificação de tipo é feita em tempo de compilação.

Linguagens Dinâmicas:

A linguagem dinâmica está ligada a habilidade da linguagem de programação permitindo que a mesma variável possa receber valores de tipos diferentes ao longo da execução do código.
